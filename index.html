أدوات كالي مرتبة وسهلة البحث

هذه الصفحة لا تعطي خطوات هجوم ولا أوامر تشغيل. الهدف: تنظيم الأدوات وفهم استخداماتها بشكل عام، ضمن تدريب قانوني مصرح به.

تنبيه قانوني: استخدم هذه المعلومات فقط على أنظمة تملكها أو لديك إذن صريح لاختبارها. أي استخدام غير مصرح به قد يكون مخالفاً للقانون.
إخلاء مسؤولية: استخدامك لهذه الصفحة ومحتواها على مسؤوليتك. لا نتحمل أي مسؤولية عن أي ضرر (مباشر أو غير مباشر) قد ينتج عن سوء الاستخدام أو تشغيل الأدوات خارج نطاقٍ مصرح به.
وضع الأمان: إخفاء الأدوات عالية الخطورة عرض الأدوات الدفاعية فقط عرض أدوات الهجوم (ثنائية الاستخدام) فقط تضمين الأدوات المستوردة

استيراد قائمة أدوات (JSON) اختياري
إذا بدك "كل أدوات كالي" فعلياً: حمّل/اجمع قائمة كاملة بصيغة JSON ثم ألصقها هنا. الصفحة بتولّد الواجهة تلقائياً بدون أوامر تشغيل.
الحقول المطلوبة: name و category. الحقول الاختيارية: risk (low|medium|high), defensive (true/false), descAr, descEn, detailsAr, detailsEn, cmdAr, cmdEn, tags.
كيف أرتّب الأدوات؟
1) صنّف الأدوات حسب "المجال" (شبكات، ويب، جنائي، هندسة عكسية...). 2) استخدم risk لتفعيل وضع الأمان. 3) اجعل الوصف عاماً وغير إجرائي (بدون خطوات أو أوامر).
الخصوصية
الاستيراد والتصدير يتمان داخل المتصفح فقط. لا يوجد إرسال بيانات لخادم.

شرح برمجة الحماية/الهجوم (تعليمي)

شرح مبسّط لكيف تُصنَّف الأدوات وكيف تعمل الفلاتر ووضع الأمان داخل هذه الصفحة (بدون خطوات هجوم أو أوامر استغلال).

هيكلة بيانات الأداة
  • الحقول المطلوبة: name و category.
  • risk يحدد مستوى الحساسية: low | medium | high.
  • defensive يميز بين دفاعي (true) وثنائي الاستخدام (false).
  • detailsAr/detailsEn نقاط شرح مفصل غير إجرائي.
  • cmdAr/cmdEn شرح “شكل الأوامر” بشكل آمن (ليس أوامر هجوم).
[
  {
    "name": "tool-name",
    "category": "Web",
    "risk": "medium",
    "defensive": true,
    "descAr": "وصف عام...",
    "detailsAr": ["نقطة 1", "نقطة 2"],
    "cmdAr": ["ملاحظات عامة عن الاستخدام"],
    "tags": ["web", "audit"]
  }
]
منطق الفلاتر (الحماية)
  • بحث النص يشمل: الاسم، التصنيف، الوصف، الشرح المفصل، وشرح الأوامر.
  • “عرض الأدوات الدفاعية فقط” يعرض defensive=true.
  • “عرض أدوات الهجوم (ثنائية الاستخدام) فقط” يعرض defensive=false.
  • فلتر التصنيف/الخطورة يحدّ النتائج حسب القيم المختارة.
if (onlyDef && !tool.defensive) skip
if (onlyDual && tool.defensive) skip
if (cat !== "all" && tool.category !== cat) skip
if (risk !== "all" && tool.risk !== risk) skip
if (q && !matchesText(tool, q)) skip
 show(tool)
دليل الأوامر (مفصل جداً)
  • اقرأ صيغة الأداة من: --help / man / help. هذه أفضل نقطة بدء لأنها تطابق نسختك.
  • فهم القالب: command [OPTIONS] <INPUT> ... ثم إخراج النتائج لملف تقرير.
  • علامات CLI الشائعة: -h/--help, -v/--verbose, -q/--quiet (قد لا تتوفر في كل الأدوات).
  • إعادة توجيه المخرجات (لينكس): > حفظ، >> إضافة، 2> للأخطاء، و2>&1 للدمج.
  • استخدم less/grep على ملفات التقارير لتصفية النتائج بدل تكرار التشغيل بلا داعٍ.
  • الصلاحيات: sudo فقط عند الحاجة (التقاط، واجهات). لا تشغّل كل شيء كـ root.
  • توثيق: احفظ (التاريخ/الإصدار/الإعدادات) لتعيد التحقق بعد الإصلاح.
# (Linux) discover built-in help
tool --help
man tool
tool help

# save outputs for review
tool [options] <input> > report.txt 2> errors.txt

# inspect large outputs safely
tool [options] <input> | less
منطق “الهجوم” هنا (Dual-use) + وضع الأمان
  • الأدوات ثنائية الاستخدام تُوسم عادةً defensive=false وقد تكون risk=high لأنها حساسة ويمكن إساءة استخدامها.
  • عند تفعيل “وضع الأمان” يتم إخفاء أي أداة risk=high افتراضياً.
  • الهدف تعليمي: عرض سياق عام فقط بدون تفاصيل تشغيلية.
if (safetyMode && tool.risk === "high") hide(tool)
 else show(tool)
أساسيات برمجة الحماية (دفاع)
  • تحقق من المدخلات وطبّق مبدأ أقل صلاحيات.
  • استخدم استعلامات مُعلّمة/Prepared Statements لتجنب حقن SQL.
  • فعّل التسجيل (Logging) والتنبيهات وراقب مؤشرات الاختراق.
  • طبّق Rate limiting وMFA وإدارة أسرار آمنة (Secrets).
validate(input)
query("SELECT ... WHERE id = ?", [id])
log(event)
أساسيات برمجة الاختبار الهجومي (محاكاة قانونية)
  • مختبر/إذن مكتوب + نطاق واضح (Scope) قبل أي اختبار.
  • ركّز على “التحقق والقياس” بدل الاستغلال: جمع أدلة، إعادة إنتاج آمنة، ثم توصيات إصلاح.
  • وثّق النتائج: الوقت، الإصدارات، الإعدادات، والـ Logs.
  • شارك تفاصيل تشغيلية أقل ما يمكن، وركّز على معالجة السبب الجذري.
Report template:
- Scope
- Observation
- Risk
- Recommendation
الاستيراد والتخزين
  • الاستيراد يتم محلياً داخل المتصفح (لا يوجد إرسال لخادم).
  • يتم حفظ الإعدادات والقائمة في localStorage لتبقى بعد إعادة فتح الصفحة.
  • عند الاستيراد، يتم دمج الأدوات بالاسم (آخر نسخة تفوز).
  • تم حذف القائمة المدمجة للأدوات حاليًا (0 أداة مدمجة).
